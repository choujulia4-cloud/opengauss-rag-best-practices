# 附录：文件命名对应关系整理

本附录用于将原始文件名与整理后文件名、配置摘要、建议目录统一起来，方便正式报告引用和后续项目归档。

## 1. 命名原则

1. 文件名优先体现“任务类型 + 核心配置 + 版本编号”。
2. 混合比较文件与单一配置实验文件分开命名。
3. 评价/评分文件与原始输出文件分开命名。
4. 报告骨架、方法论与实验结果文件分开存放，避免混淆“结论”和“证据”。

## 3. 使用建议

### 3.1 正式报告引用
正式报告中应优先引用：
- 主统计文件
- 主评分文件
- 生成器对比文件
- gold context 结果文件
- query / gold context 文件
- 核心代码文件

### 3.2 附录引用
附录中可收纳：
- 单一配置原始日志
- top-k 原始输出
- raw_output.zip
- 原始终端打印记录
- 典型报错与修复经验 raw 材料

## 4. 注意事项

1. 同一配置下的多份版本文件不应随意删并，除非确认完全重复。
2. 原文件名中的个别拼写错误（如 `pji`）在整理时可以校正，但要在附录中说明原始来源。
3. 正文应使用“整理后文件名 + 配置摘要”作为引用方式，raw 原名主要保留在附录中用于追溯。fileciteturn7file18
